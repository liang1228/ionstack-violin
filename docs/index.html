🔬 漏洞机制

GhostLock 是 Linux 内核 rt_mutex 的 PI (Priority Inheritance) 链遍历中的栈 UAF：

  1. REQUEUE — FUTEX_CMP_REQUEUE_PI 将 waiter 从 f_wait 移动到 f_pi_target
  2. UAF — owner 解锁后，waiter 获取锁，rt_mutex_cleanup_proxy_lock 跳过 remove_waiter，pi_blocked_on 指向已释放栈
  3. PI Chain Walk — consumer 调用 sched_setattr 触发 rt_mutex_adjust_prio_chain，遍历伪造的 lock/task/waiter
  4. Direct Write — pselect fd_set words 在内核栈上覆写 waiter 字段，PI chain walk 时内核写入目标值到目标地址

关键发现： violin 的 misc_fops 无 NULL 字段，rb_insert 写入原语不可用。改用 direct pselect write 路径，通过 shape 参数控制 waiter 字段对齐。

🔧 构建

# 克隆
git clone https://github.com/liang1228/ionstack-violin.git
cd ionstack-violin

# (可选) 下载 KernelSU
# https://github.com/tiann/KernelSU/releases
cp ksud-arm64 build/embed/ksud_aarch64

# 构建 violin target
make PROJECT=violin-v-oss

# 推送到设备
adb push build/violin-v-oss/bin/preload.so /data/local/tmp/
adb shell "cd /data/local/tmp && LD_PRELOAD=./preload.so /system/bin/sh"
